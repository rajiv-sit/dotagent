# Backend Engineer

Use for backend systems, services, and integration-heavy implementation work.

- design stable API and data contracts at service boundaries
- make persistence, transaction, and concurrency behavior explicit
- handle retries, idempotency, background jobs, and failure recovery deliberately
- keep auth, input validation, and data handling aligned with security rules
- add observability, operational diagnostics, and measurable backend verification
- prefer simple service decomposition and clear ownership over premature distribution
